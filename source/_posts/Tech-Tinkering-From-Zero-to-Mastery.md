# 玩机：从入门到入土

"玩机"，这个词对于中国互联网早期用户而言，承载着熟悉又陌生的记忆。它象征着技术自由、掌控权与极客精神，它是对系统臃肿的反抗，对精简系统、榨干硬件潜力的执着追求。这是用户夺回设备话语权、拒绝商业预设路径的宣言。在官方系统日益封闭和完善的当下，玩机文化虽显式微，却深深烙印着一代人追求纯粹掌控的数字记忆。那份亲手重塑设备、挑战限制、突破限制的极客精神，正是数字自由最初的微光。它代表着一种不妥协的态度：**我的设备，理应由我做主！**

## 玩机是什么？

玩机，泛指对个人持有的电子设备进行自定义的行为。
需要注意的是，玩机不仅限于安卓设备，也涵盖电脑、手表等多种设备类型。其操作形式同样广泛，既包括深度修改系统固件（ROM），也包括不更改固件的基础自定义设置。**在本系列文章中，如无特殊说明，"玩机"默认指在安卓设备上的相关操作。**
在之后的教程里，你若是碰到了困惑不解的地方都是很正常的，就像是第一次坐飞机去旅行一样，固然过程艰辛，路程颠簸，但你总是能到达最终的目的地。在本系列文章中，**我会尽可能的替你扫除盲点**，你大概可以从[知识补全](#知识补全)中寻找到你想要的答案

## 获取系统的最高权限：Root

在条件允许的情况下，获取 Root 权限是深入玩机的理想前提。但此过程通常涉及修改 ROM，因此**必须**事先解锁设备的 Bootloader 锁（BL 锁）。

### 什么是 Root？

Root 是类 Unix 系统（如 Linux、macOS）中的最高权限账户。由于安卓基于 Linux 内核，Root 权限在安卓系统中同样是最高的权限账户。
**重要提示：** 获取 Root 权限会带来潜在的安全风险（如恶意软件可能获得更高权限），并**通常会导致设备官方保修失效**。进行 Root 操作前，请务必充分了解风险并做好准备。
**注意：** 安卓系统基于 Linux 内核，但它本身并非标准的 Linux 发行版。

### 什么是 Bootloader 锁 (BL 锁)？

Bootloader (BL)，即引导加载程序，是设备启动时运行的第一个程序，负责加载并启动操作系统。Bootloader 锁 (BL 锁) 是一种由厂商设置的保护机制，旨在防止用户刷写非官方或未经认证的系统固件。它会验证启动时加载的固件签名，若校验失败，则可能导致设备无法正常启动（俗称"变砖"）。
**解锁说明：** 不同厂商设备的 Bootloader 解锁方式差异很大，具体操作流程请查阅对应设备的官方或社区指南，本文不展开详述。

### 开启 Root 之旅

首先，你需要选择一个可靠的 Root 权限管理工具，例如广受欢迎的 **Magisk** ，此处以 Magisk 为例，其它管理工具请翻阅目录自行查找。

1. **准备工作**  
   - 下载开源可靠的 Root 权限管理工具 **Magisk**（建议从[Github](https://github.com/topjohnwu/Magisk)或可信渠道获取，避免使用非正规来源文件）  
   - 获取设备对应的官方固件包（用于提取 boot.img），可通过官方下载或专业工具提取  
   - 安装 [ADB 和 Fastboot 工具](https://developer.android.com/tools/releases/platform-tools)

2. **操作流程**  
   - 从官方固件包中解压出 boot.img  
   - 将 boot.img 传入手机内部存储  
   - 打开 Magisk → 点击"安装" → 选择"选择并修补文件" → 选中 boot.img  
   - 修补后的文件将生成于：`/sdcard/Download/magisk_patched_[随机字符].img`  
   - 将此文件传回电脑，执行刷入命令：
```bash
fastboot flash boot magisk_patched_*.img
fastboot reboot
```
3. 验证结果
设备重启后，打开 Magisk 应用若显示"已安装"，即表示 Root 成功。

## 必经之地:隐藏Root
银行、支付、游戏等应用会主动扫描 Root 状态，一旦发现设备已经Root就会强制退出甚至封号，所以隐藏Root是很有必要的

1. 进入 **设置**（齿轮图标），打开 **Zygisk**
2. 下载 [Shamiko 模块](https://github.com/LSPosed/LSPosed.github.io/releases)
3. 在 Magisk 中进入 **模块** 页面，点击 **从本地安装**，选择下载的 Shamiko 模块文件(通常为zip)。
4. 等待刷入完成，然后 **重启设备**

现在，你就可以进入**设置**，找到配置排除列表，勾选该应用及其所有子进程（点击应用名称展开，确保全选），这些APP就无法检测到Root了

## 玩机的殊途同归:美化
前文有提到，玩机就是对个人持有的电子设备进行自定义的行为。虽然“自定义”的范围较广，但毫无疑问，**美化**是其中必不可少的一环。
美化的方式有很多，涵盖对字体、主题、壁纸等的更改与美化，本文将介绍几种通用性较强、效果显著的美化方式。

### 壁纸:美化的灵魂
设置壁纸并不困难，但实现**动态壁纸**往往需要借助特定工具。尽管诸如B站、抖音、快手等应用提供了视频设为壁纸的功能，但其本身较为**臃肿**，对追求系统**轻量化**和极致掌控的玩机用户而言并非最优。
在这里，笔者推荐一款软件:**[Wallpaper Engine on Android]**，中文名为**[壁纸引擎]**，读者可以从[官网](https://www.wallpaperengine.io/android/zh-hant)下载该软件
**壁纸引擎**的核心优势在于：
1.  **轻量高效**：专注于壁纸功能，资源占用低。
2.  **格式支持广**：支持视频文件及其专属 `.mpkg` 等格式壁纸。
3.  **生态互联**：若您已在PC端购买Steam版的Wallpaper Engine，可通过本地网络**无缝同步**电脑上精心收藏的壁纸到移动设备，构建统一的视觉体验。

### 字体美化：清晰悦读与个性表达
字体是系统UI的基础元素，更换字体能显著改变视觉风格。以下推荐几类成熟的美化方案，**请注意模块兼容性因设备和系统版本而异，刷入前请务必查阅相关说明**。

## 知识补全
### 什么是`boot.img`？
为什么Root要修改它？`boot.img`是系统启动分区镜像，包含内核和初始程序。Root需在此分区注入权限管理代码（如Magisk），因此需修改并刷入此文件。

## ADB和Fastboot是什么？如何安装？ 
- **ADB** (Android Debug Bridge)：通过电脑调试手机的**命令行工具**（如传输文件、安装应用）  
- **Fastboot**：**刷机模式工具**（用于刷入分区镜像）  
安装步骤：  
1. 访问[Android开发者网站](https://developer.android.com/tools/releases/platform-tools)下载工具包  
2. 解压后，将文件夹路径添加到系统环境变量`PATH`中
3. 终端输入 `adb version` 或 `fastboot --version` 验证是否成功

### 如何进入Fastboot模式？
通常操作：
1. 关闭手机  
2. 长按 **电源键 + 音量减键**（部分机型是**电源键 + 音量加键**）  
3. 看到屏幕显示**Fastboot**或**Bootloader**字样即成功
当然，各个品牌进入 Fastboot 的操作不一，读者可查看[各品牌进入Fastboot方式表](#各品牌进入Fastboot方式表)

### 什么是Zygisk？和Riru有什么区别？ 
- **Zygisk**：Magisk内置的**注入机制**（需Magisk v24+），直接集成在系统进程  
- **Riru**：旧版依赖的**独立模块**（已逐步淘汰）  
**优先使用Zygisk**（更高效且兼容新系统）。

### 模块刷入后需要重启才能生效吗？
需要！**任何Magisk模块安装后必须重启**，因为修改涉及系统底层进程。

### 模块在哪里下载？ 
 - 推荐来源：[酷安](https://www.coolapk.com/)、[GitHub](https://github.com/)、[Lsposed国内官网](https://lsposed.cn/)

 - 建议优先下载开源的模块
 
### Recovery模式和Fastboot模式有什么区别？ 
| **模式**       | **功能**                     | 进入方式                  |  
|----------------|------------------------------|--------------------------|  
| **Recovery**   | 恢复/刷入完整ROM包           | 关机后按 **电源+音量加键** |  
| **Fastboot**   | 刷入分区镜像（如boot.img）   | 关机后按 **电源+音量减键** |
